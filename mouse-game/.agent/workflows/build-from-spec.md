# 명세서 기반 개발 워크플로우

## 1. 명세서 확인
- `docs/spec.md` 파일을 읽고 요구사항을 파악한다.
- 명세서에 기능 목록, UI 레이아웃, 동작 흐름이 정의되어 있어야 한다.

## 2. 구현 계획 수립
- 명세서를 바탕으로 `implementation_plan.md` 아티팩트를 작성한다.
- 컴포넌트별로 나누어 구현 순서를 정한다.
- 유저 리뷰를 받은 후 진행한다.

## 3. 구현
- 명세서의 각 항목을 순서대로 구현한다.
- PySide6 컨벤션을 따른다:
  - `QMainWindow`를 메인 윈도우로 사용
  - 시그널-슬롯 패턴으로 이벤트 처리
  - `setStyleSheet()`로 스타일링
  - 레이아웃 매니저(`QVBoxLayout`, `QHBoxLayout`, `QGridLayout`)로 UI 배치

## 4. 검증
- 앱을 실행하여 명세서의 각 항목이 구현되었는지 확인한다.
- `/run` 워크플로우로 실행한다.

## 5. README 업데이트
- 작업 완료 후 `README.md` 파일을 현재 프로젝트 상태에 맞게 업데이트한다.
- 새로운 기능, 변경된 구조, 사용법 등을 반영한다.

## 명세서 템플릿

명세서는 `docs/spec.md`에 아래 형식으로 작성한다:

```markdown
# [프로그램 이름]

## 개요
프로그램의 목적과 간단한 설명

## 기능 목록
- [ ] 기능 1: 설명
- [ ] 기능 2: 설명

## UI 레이아웃
화면 구성 설명 (텍스트 또는 다이어그램)

## 동작 흐름
1. 사용자가 A를 하면
2. 프로그램이 B를 수행
3. 결과 C를 표시

## 기술 요구사항
- Python 3.14+
- PySide6 6.10+
```
